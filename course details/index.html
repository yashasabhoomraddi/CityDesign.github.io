2. Introduction

As urbanization continues to surge, the demand for efficient and sustainable public transportation systems becomes also grows in tandem. As cities expand, infrastructure often struggles to keep pace with the growing population, leading to traffic congestion, air pollution, and inefficient public transportation systems. These issues not only hinder economic growth but also reduce the quality of life for residents. Among these challenges, urban mobility remains a critical concern, as existing systems fail to cater to the increasing demands of daily commuters. This project focuses on designing a smart city with a special emphasis on innovative solutions for its subway systems and networks. Reading the white paper has made it clear that there is an immediate need for solutions that would change the world for the better.

One of the prominent problems people are facing today is the ease of travel. Solving these urban challenges is essential to creating cities that are resilient and efficient. This project focuses on designing a smart city with advanced subway systems, using modern technologies to provide sustainable and effective transportation solutions. By reducing traffic congestion, cutting down carbon emissions, and improving access to public transport, the project aims to tackle the critical problems caused by urbanization. It seeks to promote sustainable urban growth while addressing key societal issues such as mobility and accessibility disparities.

3. Problem Definition

Implementation of an efficient Subway System: Amongst the different modes of travel present today, railways and/or subway system is one of the cheapest and most frequently chosen mode of transport by the public. Today in India we face a myriad of problem in the railways system, be it overcrowding due to improper scheduling of trains or lack of facilities offered to the public waiting to board one. Efficient commute is the need of the hour and that provided us with the impetus to work on resolving this issue. The domain of subway systems and networks involves the design, optimization, and management of rapid transit systems. It includes key aspects such as route planning, real-time monitoring, scheduling, passenger experience enhancement, etc.

4. Functionality Selection

Functionality Name Known Unknown Principles Applicable Algorithms Data Structures
Predictive Maintenance The module predicts train maintenance requirements based on their age, birth year, and predefined maintenance intervals. Static schedules may not account for factors like usage intensity or unforeseen issues. Decomposition, Pattern-recognition, Pre-computing Quick Sort Vectors, Structures, Arrays
Hygiene Management The module is designed to collect and log hygiene-related complaints for subway stations. Limited complaint types and future scalability challenges. Decomposition, Pattern Recognition, Abstraction Merge Sort Vectors, Strings, Structures

5. Functionality Analysis

1) Predictive Maintenance:

Step Description
Initialize File Path Dynamically determines the file path to store complaints.
Collect Station Number Prompts the user to input the subway station number.
Display Complaint Options Displays predefined hygiene-related complaints and collects the user's choice.
Validate Input Ensures the user's input is valid (within the range of complaint options).
Log Complaint Opens the file, writes the complaint details in a formatted way, and closes the file.
Provide Feedback Informs the user whether the complaint was logged successfully or if an error occurred.

2) Hygiene Maintenance:

Train Maintenance Workflow
Step Description
Initialize Train Data A list of 20 predefined train objects is created, each with its attributes (name, age, birth year).
Quick Sort Execution Trains are sorted based on their next maintenance year using Quick Sort.
Input Train Number The user is prompted to select a train number (1-20).
Input Validation Checks if the train number is within the valid range.
Next Maintenance Calculation The next maintenance year is calculated using the train's birth year and the rule of 5-year intervals.
Maintenance Status Update Flags trains that require immediate maintenance based on the current year and next maintenance year.
Display Details Outputs the details of the selected train and all trains sorted by their next maintenance year.
